LivenessError

case class LivenessError[Event](monitorName: String, propertyName: String, explanation: String, errorTrace: ErrorTrace[Event]) extends Error[Event] with Product with Serializable

A liveness error represents an error caused by a certain event not occurring in the trace.

An example of a property which can cause a liveness error is the following:


class Requirement extends Monitor[Event] {
  requirement('CommandMustSucceed) {
    case COMMAND(x) =>
      hot {
        case SUCCESS(x) => ok
      }
  }
}


Suppose for example that a COMMAND("A") is observed, but no SUCCESS("A") is observed thereafter. In this case a liveness error will be produced.

monitorName

name of monitor in which property resides.

propertyName

name of property violated.

explanation

text provided by the informal method in class tracecontract.Monitor. Provides an informal explanation of the property.

errorTrace

the sequence of events that lead to the error. For a liveness property typically the event(s) that caused the expectation of the missing event.
